# テスト方針

## テストの種類

- **ユニットテスト**: `tests/` に配置。コアロジックを中心にテスト
- **統合テスト**: LINE API等の外部サービスはモックで代替

## 実行

```bash
uv run pytest              # 全テスト
uv run pytest -x           # 最初の失敗で停止
uv run pytest -k "keyword" # 特定テストのみ
```

## 原則

- コアロジック（`core/`）は必ずテストを書く
- アダプター層（`adapters/`）は外部APIをモックしてテスト
- テストが通らない状態でコミットしない
