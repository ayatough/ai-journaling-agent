# 定期同期スキル（Loop用）

`/loop` で定期実行される。1日1〜2回。

## 実行内容

1. **Issue確認**: アサイン済みIssueがあれば処理（仕様明確なもののみ）
2. **実験ログ確認**: `outputs/experiments_log.md` に新しいエントリがあれば確認
3. **ADR判断**: 重要な設計判断があればADR化を提案（人間の承認待ち）
4. **進捗報告**: 実施した内容を簡潔に報告

## 禁止事項

- GitHub Discussionsへの書き込み（人間専用）
- 仕様が曖昧なIssueの自動実装
- テストが通らない状態でのコミット
