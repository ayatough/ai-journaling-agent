# ADR-0002: ユーザーインターフェースにLINE Messaging APIを選択

## ステータス
承認

## 背景
ジャーナリングアプリのMVPで、ユーザーとAIが対話するインターフェースの選定が必要。テキストと画像の送受信が必須要件。

## 選択肢
1. **Slack**: 開発者に馴染み深いが、一般ユーザーには普及率が低い
2. **LINE**: 日本のユーザーに最も普及。リッチメニュー・クイックリプライ等のUI機能が充実
3. **Discord**: 開発者向け。一般ユーザーへの展開が難しい
4. **メール**: プラットフォーム非依存だがリアルタイム性が低い

## 決定
**LINE Messaging API**を選択。

理由:
- 日本のユーザーに最も馴染みがあり、日常的に使うアプリ上で記録ができる
- リッチメニューでLevel選択をワンタップ化、クイックリプライで絵文字選択が可能
- 画像メッセージの送受信に対応（写真アップロード要件を満たす）
- 将来のアプリ化時もLINE連携は維持可能

## 影響
- LINE Developers登録・チャネル設定が必要
- Webhook受信用のサーバー（ngrok等）がMVP段階で必要
- アダプター層で分離することで、将来の独自アプリUIへの移行も可能
