# ADR-0001: MVP段階の言語としてPythonを選択

## ステータス
承認

## 背景
バックエンドの言語として最終的にはRustを目指すが、MVP段階でどの言語を使うかの判断が必要。

## 選択肢
1. **最初からRust**: 書き直し不要だが、プロトタイピング速度が落ちる
2. **Python → Rust移行**: MVPを素早く検証し、本番化時にRustへ移行

## 決定
**Python → Rust移行**を選択。

理由:
- LINE Bot SDK等のエコシステムが成熟しており、MVP検証が速い
- コア/アダプター分離により、コアロジックのRust移植が容易
- MVP段階では仕様の変更が頻繁に起きるため、動的言語の柔軟性が有利

## 影響
- 移行コストが発生するが、コア/アダプター分離で最小化
- Python 3.12+の型ヒントを活用し、Rust移行時の型設計の参考にする
